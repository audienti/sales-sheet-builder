# Research And Evidence Rules

Use this reference when gathering source material before writing a sales sheet.

## Source URL Extraction Checklist

Extract what is present. If something is vague, mark it as `Weak` or `Inference`; if missing, mark it as `Missing`.

- Company name
- Product or service name
- Plain-language product category
- Target audience
- Buyer persona
- Economic buyer
- Daily user
- Pain owner
- Main pain points
- Cost of pain
- Product promise
- Claimed outcomes
- Features
- Benefits
- Differentiators
- Use cases
- Company proof points
- Testimonials
- Case studies
- Metrics
- Screenshots or workflow examples
- Contact or conversion path
- Common language used on the page
- Current tools, methods, services, or workarounds the buyer likely uses
- Status quo alternative
- Reference competitors named or implied
- Economic competitors named or implied
- Job executor
- Core functional job
- Current job steps
- Desired outcomes
- Related jobs
- Emotional or social jobs
- Consumption-chain jobs

## Gap Analysis Elements

Score each item as `Present`, `Weak`, or `Missing`.

| Sales sheet element | What to check |
| --- | --- |
| Industry facts or figures | Credible evidence that explains the pain, market, urgency, or category |
| Clear pain | Buyer-recognizable problem in plain language |
| Cost of pain | What gets worse if the buyer keeps the current process |
| Promise | Specific change the product claims to create |
| Product category | Plain category the buyer already understands |
| Product definition | One-sentence definition of what it is, who it helps, how, and why |
| How it works | Simple mechanism, no technical deep dive |
| Benefits | Why the buyer cares |
| Outcomes | What changes for the business |
| Company proof | Customer results, testimonials, case studies, usage data, screenshots, named examples |
| Industry proof | External evidence validating the pain or solution category |
| Differentiation | Difference versus the current alternative |
| Alternative map | Status quo, reference competitors, and economic competitors |
| JTBD map | Job executor, core functional job, job steps, desired outcomes, related jobs, emotional or social jobs, and consumption-chain jobs |
| Four-question pitch | Increase/decrease goal, current approach, downsides, and new way |
| Use cases | Concrete moments where the buyer would use it |
| Objections | Likely reason the buyer hesitates |
| Buyer-specific language | Words the buyer would naturally use |
| Sales conversation hook | Reasonable next conversation prompt |

For each weak or missing item, state what research, rewrite, or asset would improve the sheet.

## Proof Hierarchy

Use company proof first:

- Customer results
- Testimonials
- Case studies
- Pilot metrics
- Product usage data
- Screenshots
- Before-and-after examples
- Named customer examples
- Specific claims on the source page

If company proof is absent, use external proof to validate the solution approach, not to pretend the company has results.

Good external proof validates:

- The pain is real
- The market is moving in this direction
- This type of solution can address the pain
- The buyer has reason to act now
- The outcome matters to the business

Prefer credible sources:

- Gartner, Forrester, McKinsey, Deloitte, PwC
- Salesforce, HubSpot, LinkedIn, or credible category vendors with primary research
- Academic research
- Government data
- Trade association data
- Relevant industry reports

Use current sources for buyer behavior, sales performance, AI adoption, market size, regulation, technology, or economic claims. Avoid weak blogs unless no stronger source exists.

## Evidence Log Fields

Create an evidence log before writing the final sheet.

| Field | Rule |
| --- | --- |
| Claim | The exact claim being considered |
| Source | Page, report, article, case study, or dataset name |
| Source type | Company page, customer proof, analyst report, vendor research, government data, academic source, etc. |
| Date | Publication date or access date if no publication date exists |
| URL | Link to the source |
| Support | How the claim supports the sales sheet |
| Confidence | `High`, `Medium`, or `Low` |

Confidence levels:

- `High`: directly supported by the source.
- `Medium`: reasonable interpretation based on credible evidence.
- `Low`: useful context, but indirect support.

Do not use low-confidence claims in the final sheet unless clearly framed as context.

## Research Discipline

- Do not include uncited numbers in the final sales sheet.
- Do not convert category evidence into company-specific proof.
- Do not use a competitor's claim as proof for the subject company.
- If sources conflict, use the more primary or recent source and note the uncertainty.
- If the proof base is thin, make the final sales sheet more modest instead of filling gaps with hype.
